# План: практические занятия для разделов 01–08

Дата: 2026-07-15

## Контекст

- Структура репозитория: 8 содержательных разделов (`01-08`) + `09-glossary`
(справочник, без опросника/повествования — исключён из практики).
- `AGENTS.md` требует не плодить файлы/директории без необходимости, но
практические занятия — это новый тип контента (код, а не проза), поэтому
выделение отдельной директории оправданно.
- Решения по открытым вопросам:
  1. Практика живёт в `exercises/NN-.../` — зеркалирует нумерацию разделов,
  
    не смешивается с прозой в `01-08/README.md`.
  2. Каждое упражнение = **заготовка + эталонное решение** (`exNN_*.py` +
  
    `exNN_*_solution.py`).
  3. Для LLM-упражнений используется **LM Studio** как OpenAI-совместимый
  
    локальный сервер (`http://localhost:1234/v1`) через библиотеку `openai`.
- Каждый раздел README уже даёт основу для практики: примеры кода, псевдокод
пайплайнов, таблицы «что выбрать».
- Раздел 06 (LoRA/квантизация/DPO) не требует реальной LLM — легковесные демо
на toy-тензорах (PyTorch/NumPy), без скачивания моделей.
- Вариант хранения практики: `exercises/NN-.../` (централизованная директория
верхнего уровня, по аналогии с `diagrams/`), а не `NN-.../exercises/` внутри
каждого раздела — решение зафиксировано и не пересматривается.
- Виртуальное окружение для упражнений — **uv + Python 3.13**. Управление
зависимостями через `pyproject.toml` + `uv.lock` (не голый `pip` и не просто
`requirements.txt`), т.к. `uv` — современный стандарт для быстрого и
воспроизводимого окружения в Python-проектах.

## Цель

Добавить практические занятия к каждому из разделов 01–08: одна новая
директория `exercises/`, зеркалирующая нумерацию разделов, с заданиями на
Python, использующими локальные модели через LM Studio там, где нужен
реальный LLM-инференс.

## Шаги

### Шаг 0 — Скелет и общая инфраструктура

- Создать `exercises/README.md` с двумя явными частями:
  1. **Подготовка окружения (uv + Python 3.13)** — пошаговая инструкция:
    - установка `uv` (ссылка на официальный установщик, одна команда);
    - `uv python install 3.13` (если нужной версии ещё нет);
    - `uv sync` из директории `exercises/` — создаёт `.venv` и ставит все
    зависимости по `pyproject.toml`/`uv.lock`;
    - как запускать упражнения через `uv run python 01-foundations/ex1_neuron.py`
    (без ручной активации `.venv`, но с упоминанием `source .venv/bin/activate`
    как альтернативы);
    - как добавлять новую зависимость при доработке упражнений: `uv add <pkg>`.
  2. **Подготовка LM Studio** — установка, запуск локального сервера (порт
  
    1234), какую chat-модель и какую embedding-модель загрузить (например
     `nomic-embed-text` для эмбеддингов, любая небольшая chat-модель),
     проверка доступности сервера.
  - Также: общая конвенция файлов (`exNN_*.py` = заготовка с `TODO`,
  `exNN_*_solution.py` = эталон, оба содержат `if __name__ == "__main__":`
  с самопроверкой через `assert`/печать результата), таблица-хаб со
  ссылками на практику всех 8 разделов.
- Создать `exercises/pyproject.toml` (`requires-python = ">=3.13,<3.14"`,
зависимости: `openai`, `numpy`, `torch`, `langchain`, `langchain-openai`,
`langgraph`, `requests`) и сгенерировать `exercises/uv.lock` через
`uv lock`. Добавить `exercises/.python-version` со значением `3.13`.
- Создать `exercises/common/lmstudio_client.py` — единая точка конфигурации
(`BASE_URL = "http://localhost:1234/v1"`, `get_client()` на `openai.OpenAI`,
понятная ошибка «LM Studio не запущен» при `ConnectionError`).
- **Зависимости:** ничего, можно делать первым.

### Шаг 1 — `exercises/01-foundations/`

- **ex1_neuron.py**: реализовать нейрон (`z = wx+b`, ReLU) и функцию
активации вручную, сравнить с/без нелинейности на игрушечном примере (без
LLM, чистый Python/NumPy) — опирается на §2–3 раздела.
- **ex2_training_loop.py**: реализовать цикл обучения линейной регрессии
(forward → loss → backward через `numpy`-градиент вручную или
`torch.autograd`) на синтетических данных, продемонстрировать overfitting
на маленьком датасете — опирается на §5 раздела.
- LLM не требуется.

### Шаг 2 — `exercises/02-llm/`

- **ex1_tokens_and_embeddings.py**: получить эмбеддинги через LM Studio
(`/v1/embeddings`) для набора слов/фраз, посчитать `cos_sim` (код уже есть
в §3 README как основа), построить наивный «ближайший сосед» — опирается
на §2–3.
- **ex2_roles_and_temperature.py**: собрать `messages` с ролями
system/user (по образцу §8), сделать несколько запросов с разной
`temperature`, сравнить разброс ответов — опирается на §5, §8.
- Требует LM Studio (chat + embedding модель).

### Шаг 3 — `exercises/03-agents/`

- **ex1_tool_calling.py**: описать 1–2 инструмента (калькулятор,
`get_weather`-заглушка) через OpenAI tool-calling схему, отправить в LM
Studio, вручную выполнить вызов и вернуть `tool` message — опирается на §2
(готовый пример в README).
- **ex2_react_loop.py**: собрать полноценный ReAct-цикл (Thought → Action →
Observation) по скелету из §3 README, с реальными вызовами модели через LM
Studio — опирается на §3.
- Требует LM Studio с моделью, поддерживающей tool calling.

### Шаг 4 — `exercises/04-rag/`

- **ex1_mini_rag_pipeline.py**: полный мини-RAG на 5–10 коротких документах:
чанкинг → эмбеддинг через LM Studio → поиск top-K (`cos_sim`, как в §3/§5
README) → сборка промпта → генерация ответа через LM Studio.
Дополнительное задание внутри файла: продемонстрировать разницу
«с RAG» / «без RAG» на вопросе о вымышленном факте.
- Требует LM Studio (chat + embedding).
- **Зависимость:** удобно опираться на `common/lmstudio_client.py` из Шага 0.

### Шаг 5 — `exercises/05-pipelines-frameworks/`

- **ex1_langchain_chain.py**: собрать `ChatPromptTemplate | ChatOpenAI` с
`base_url` на LM Studio (по образцу §3 README), сделать 2–3-шаговую
цепочку.
- **ex2_langgraph_agent.py**: реализовать граф с условным ребром (узел
«думать» → «инструмент» → назад), по скелету §4 README, подключить к LM
Studio.
- **ex3_pytorch_training.py**: реальный шаг обучения на PyTorch
(`loss.backward()`, `optimizer.step()`) — без LLM, прямое расширение
примера §6 README и связка с 01/ex2.
- Требует: `langchain`, `langchain-openai`, `langgraph`, `torch`.

### Шаг 6 — `exercises/06-model-adaptation/`

- **ex1_quantization_sim.py**: сгенерировать случайную матрицу весов
(float32), квантизовать в int8 вручную (NumPy), измерить сжатие и ошибку
восстановления — без LLM, иллюстрирует §3.
- **ex2_lora_toy.py**: на игрушечном `nn.Linear` реализовать LoRA-обёртку
(`W_frozen + B@A`), сравнить число обучаемых параметров full fine-tune vs
LoRA — опирается на пример §2 README, без скачивания реальных LLM.
- **ex3_dpo_loss.py**: реализовать формулу DPO loss на PyTorch с игрушечными
log-вероятностями пар «предпочтительный/непредпочтительный» ответ —
опирается на §5.
- LLM не требуется (сознательно легковесно, без GPU/скачивания весов).

### Шаг 7 — `exercises/07-inference/`

- **ex1_latency_vs_throughput.py**: замерить TTFT и tokens/sec на одном
запросе к LM Studio, затем — throughput при нескольких параллельных
запросах (`concurrent.futures`/`asyncio`), сравнить метрики — опирается на
§2–4.
- **ex2_streaming_ttft.py**: сравнить стриминг vs нестриминг вызов (по
образцу §5 README `model.stream(...)`), измерить воспринимаемую задержку
до первого токена.
- Требует LM Studio (chat, streaming endpoint).

### Шаг 8 — `exercises/08-safety/`

- **ex1_guardrails.py**: реализовать input/output guardrail-функции
(запрещённые паттерны, PII-фильтр на regex) вокруг вызова LM Studio, по
образцу §2 README.
- **ex2_prompt_injection.py**: смоделировать косвенную инъекцию (документ с
«спрятанной» инструкцией) в мини-RAG-контексте (переиспользовать
`04-rag/ex1`), показать до/после применения guardrail — опирается на §3.
- **ex3_mini_evals.py**: собрать маленький eval-датасет (5–10 Q/A пар),
прогнать через LM Studio, оценить точным совпадением + LLM-as-a-judge
(второй вызов той же модели) — опирается на §5.
- Требует LM Studio.

### Шаг 9 — Синхронизация документации (по правилам `AGENTS.md`)

- В корневом `README.md`: добавить пункт в «Как пользоваться» с указанием на
`exercises/` и, при необходимости, колонку/сноску в карте разделов.
- В каждом `NN-*/README.md`: добавить короткую строку-ссылку на
соответствующую практику (`Практика: ../exercises/NN-.../README.md`) рядом
с навигацией «Назад/Дальше», не трогая основной текст и порядок изложения.
- В `AGENTS.md`: зафиксировать новую конвенцию (структура `exercises/`, пара
«заготовка + solution», подход к LM Studio, окружение **uv + Python 3.13**
для `exercises/`) — чтобы будущие агенты следовали ей и не «изобретали»
новый формат или другой пакетный менеджер.
- **Зависимость:** делать после того, как структура `exercises/` создана
(Шаги 0–8), чтобы ссылки были на существующие файлы.

## Риски и как их снимать


| Риск                                                                        | Митигация                                                                                                                                                                                        |
| --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Пользователь без LM Studio не сможет прогнать LLM-упражнения                | Все LLM-скрипты должны падать с понятным сообщением («Запустите LM Studio на localhost:1234 и загрузите модель») через `common/lmstudio_client.py`, а не глухим traceback                        |
| В LM Studio может не быть загруженной embedding-модели                      | В README `02-llm` и `04-rag` явно указать, какую embedding-модель загрузить (например `nomic-embed-text` / `text-embedding-bge`)                                                                 |
| Раздел 06 может скатиться в «слишком абстрактно, без практики»              | Использовать toy-тензоры (не настоящие веса LLM) — быстро выполняется на CPU, но иллюстрирует именно механику (LoRA-декомпозиция, квантизация, DPO loss), без GPU/скачивания гигабайтных моделей |
| Дублирование логики между `01/ex2` и `05/ex3` (PyTorch training loop)       | Явно развести: 01 — чистый Python/NumPy без autograd (концептуально), 05 — реальный PyTorch с `autograd`/`optimizer` (инженерно), с перекрёстной ссылкой в README                                |
| Несогласованность структуры между разделами (кто-то забудет `_solution.py`) | Зафиксировать единый шаблон файла в `exercises/README.md` и следовать ему во всех 8 разделах                                                                                                     |
| Раздувание контекста/времени на реализацию (8 разделов × 2–3 упражнения)    | Можно распараллелить по разделам как независимые подзадачи (каждый раздел самодостаточен после Шага 0)                                                                                           |

## Критерии готовности

- `exercises/README.md` содержит полную инструкцию по подготовке окружения:
установка `uv`, `uv python install 3.13`, `uv sync`, запуск упражнений
через `uv run`, добавление зависимостей через `uv add` — воспроизводимо
с нуля на чистой машине.
- `exercises/pyproject.toml` фиксирует `requires-python = ">=3.13,<3.14"`;
`exercises/uv.lock` закоммичен и соответствует `pyproject.toml`
(`uv lock --check` проходит без изменений).
- Каждый из 8 разделов имеет `exercises/NN-.../README.md` с описанием
заданий и ссылкой на исходный раздел.
- Каждое упражнение — пара файлов (заготовка с `TODO` + `_solution.py`), оба
синтаксически валидны (`uv run python -m py_compile <файл>`).
- Не-LLM упражнения (01, 05/ex3, 06) выполняются локально без внешних
сервисов через `uv run` и проходят свои `assert`-проверки.
- LLM-упражнения используют только `common/lmstudio_client.py` /
`base_url=http://localhost:1234/v1`, не хардкодят ключи OpenAI, и
корректно сообщают об ошибке при недоступном сервере.
- Корневой `README.md` и `AGENTS.md` синхронизированы с новой конвенцией
практики (включая упоминание uv + Python 3.13 как окружения для `exercises/`).
