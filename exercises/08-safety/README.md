# Практика 08 — Безопасность и надёжность

Опирается на теорию: [08-safety](../../08-safety/README.md).
**Требуется LM Studio** (chat; для ex2 — ещё и понимание mini-RAG из
практики 04). Функции-гардрейлы и метрики проверяются и оффлайн.

## Упражнения

### ex1_guardrails — ограничители входа и выхода

Реализуете input-guardrail (блок инъекционных/запрещённых паттернов) и
output-guardrail (маскирование PII regex-ом) вокруг вызова модели (§2).
Guardrails не делают модель умнее — ограничивают последствия.

```bash
uv run python 08-safety/ex1_guardrails/work.py
```

### ex2_prompt_injection — косвенная инъекция в RAG

Берёте mini-RAG (практика 04) и подкладываете документ со **спрятанной**
инструкцией (§3). Защищаетесь двумя слоями: очистка контекста
(`sanitize_context`) + пометка контекста как **недоверенных данных** в
промпте. Показываете разницу до/после guardrail.

```bash
uv run python 08-safety/ex2_prompt_injection/work.py
```

### ex3_mini_evals — мини-eval-набор

Собираете eval-датасет из 5 Q/A-пар (§5), прогоняете через модель и
оцениваете двумя метриками: **exact match** и **LLM-as-a-judge** (второй
вызов той же модели). Видите, почему точного совпадения недостаточно.

```bash
uv run python 08-safety/ex3_mini_evals/work.py
```

> Как запускать упражнения (`setup_practice.py`, `work.py`, `solution.py`) —
> см. [общий README практики](../README.md#3-конвенция-файлов).

> Все три `solution.py` проверяют «чистые» функции (гардрейлы, санитайзер,
> exact_match) оффлайн, даже если LM Studio недоступен.
