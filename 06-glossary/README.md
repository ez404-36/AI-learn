# 06. Глоссарий

Алфавитный справочник всех терминов базы знаний. Каждая запись — краткое определение и ссылка на раздел, где термин разобран подробно.

Используйте его для быстрого повторения или когда встретили незнакомое слово и хотите вспомнить суть, не перечитывая весь раздел.

## Латиница

| Термин | Кратко | Примеры | Подробно |
|--------|--------|---------|----------|
| **ACP** | Протокол общения между агентами (делегирование задач, обмен результатами) | Оркестратор передаёт задачу агенту-исполнителю | [03](../03-agents/README.md#7-acp--протокол-общения-между-агентами) |
| **Agentic RAG** | RAG, где агент сам решает, что и когда искать | Агент переформулирует запрос и ищет повторно | [04](../04-rag/README.md#9-типичные-проблемы-и-улучшения) |
| **AI** | Искусственный интеллект — любая попытка сделать компьютер «умным» | Шахматный движок, голосовой ассистент, ChatGPT | [01](../01-foundations/README.md#1-ai-ml-и-deep-learning) |
| **Attention** | Механизм внимания: учёт связей между всеми токенами в трансформере | Связывает «он» с нужным существительным | [02](../02-llm/README.md#4-трансформер-и-механизм-внимания) |
| **Autograd** | Автоматическое вычисление градиентов для backpropagation (в PyTorch) | `loss.backward()` | [05](../05-pipelines-frameworks/README.md#6-pytorch-уровнем-ниже) |
| **Backpropagation** | Алгоритм вычисления, как подправить веса для уменьшения ошибки | Шаг обучения нейросети | [01](../01-foundations/README.md#4-что-значит-обучить-модель) |
| **Base model** | Базовая модель после pretraining: эрудированная, но не следует инструкциям | Llama до instruct-дообучения | [02](../02-llm/README.md#7-как-обучают-llm) |
| **Chain (цепочка)** | Линейная последовательность LLM-шагов | Промпт → LLM → парсер → следующий промпт | [05](../05-pipelines-frameworks/README.md#2-llm-пайплайн-и-цепочки-chains) |
| **Chain-of-Thought (CoT)** | Приём: попросить модель рассуждать пошагово | «Думай шаг за шагом» | [02](../02-llm/README.md#8-промпт-и-его-части) |
| **Chunk (чанк)** | Фрагмент документа — единица индексации и поиска в RAG | Абзац или ~500 токенов | [04](../04-rag/README.md#7-чанкинг-как-резать-документы) |
| **Context window** | Контекстное окно — максимум токенов за один запрос | GPT-4o — 128K, Claude — 200K | [02](../02-llm/README.md#6-контекстное-окно) |
| **Deep Learning** | Глубокое обучение — ML на основе многослойных нейросетей | Распознавание лиц, перевод, LLM | [01](../01-foundations/README.md#1-ai-ml-и-deep-learning) |
| **Embedding** | Представление текста вектором чисел; близкий смысл → близкие векторы | `text-embedding-3`, BGE, E5 | [02](../02-llm/README.md#3-embeddings-текст-как-числа) |
| **Few-shot** | Приём: дать в промпте примеры «вход → выход» | 2-3 примера перевода перед запросом | [02](../02-llm/README.md#8-промпт-и-его-части) |
| **Fine-tuning** | Дообучение модели под задачу (меняет веса) | Модель под юридические документы | [02](../02-llm/README.md#7-как-обучают-llm) |
| **Function calling** | См. Tool calling | `{ "tool": "get_weather", ... }` | [03](../03-agents/README.md#2-инструменты-и-tool-calling) |
| **Hybrid search** | Сочетание семантического и ключевого поиска | Вектора + BM25 | [04](../04-rag/README.md#9-типичные-проблемы-и-улучшения) |
| **Inference** | Использование обученной модели для предсказания | ChatGPT отвечает на вопрос | [01](../01-foundations/README.md#5-inference-использование-обученной-модели) |
| **Knowledge cutoff** | Дата, на которой «заморожены» знания модели | Модель не знает свежих событий | [02](../02-llm/README.md#9-галлюцинации-и-ограничения) |
| **LangChain** | Фреймворк с готовыми компонентами для LLM-приложений | Чат-бот с памятью и инструментами | [05](../05-pipelines-frameworks/README.md#3-langchain) |
| **LangGraph** | Библиотека для агентов и процессов в виде графа | Агент с циклом «думать → действовать» | [05](../05-pipelines-frameworks/README.md#4-langgraph) |
| **LlamaIndex** | Фреймворк, заточенный под RAG | Индексация документов и запрос к ним | [05](../05-pipelines-frameworks/README.md#5-llamaindex-и-другие) |
| **LLM** | Большая языковая модель — предсказывает следующий токен | GPT-4o, Claude, Gemini, DeepSeek, Llama | [02](../02-llm/README.md#1-что-такое-llm-в-одном-предложении) |
| **Loss** | Функция потерь — мера ошибки модели | MSE, cross-entropy | [01](../01-foundations/README.md#4-что-значит-обучить-модель) |
| **MCP** | Протокол подключения инструментов и данных к приложению с LLM («USB для AI») | Подключить GitHub-сервер к Cursor | [03](../03-agents/README.md#6-mcp--протокол-подключения-инструментов) |
| **ML** | Машинное обучение — компьютер выводит правила из примеров | Фильтр спама, рекомендации | [01](../01-foundations/README.md#1-ai-ml-и-deep-learning) |
| **Overfitting** | Переобучение — модель «зазубрила» данные, плохо обобщает | 100% на обучении, 60% на новых данных | [01](../01-foundations/README.md#4-что-значит-обучить-модель) |
| **Pretraining** | Базовое обучение на огромном корпусе текстов | Обучение GPT на текстах из интернета | [02](../02-llm/README.md#7-как-обучают-llm) |
| **Progressive disclosure** | Поэтапная загрузка Skill (метаданные → тело → ресурсы) ради экономии контекста | Сначала только `description`, тело — при срабатывании | [03](../03-agents/README.md#5-skills--переиспользуемые-наборы-навыков) |
| **PyTorch** | Библиотека для создания и обучения нейросетей | Обучение своей нейросети с нуля | [05](../05-pipelines-frameworks/README.md#6-pytorch-уровнем-ниже) |
| **RAG** | Поиск релевантных фрагментов + подстановка в промпт перед генерацией | Чат-бот по документации компании | [04](../04-rag/README.md#2-rag-в-одном-предложении) |
| **ReAct** | Паттерн агента «рассуждение → действие → наблюдение» в цикле | Думать → искать → читать → ответить | [03](../03-agents/README.md#3-цикл-работы-агента-react) |
| **Re-ranking** | Точная пересортировка найденных в RAG кандидатов | Cohere Rerank, cross-encoder | [04](../04-rag/README.md#9-типичные-проблемы-и-улучшения) |
| **Retrieval** | Этап поиска релевантных фрагментов в RAG | Найти 5 нужных абзацев из 1000 | [04](../04-rag/README.md#5-этап-запроса-online-rag-пайплайн) |
| **RLHF** | Дообучение по человеческим оценкам ответов | Так из GPT-3 сделали ChatGPT | [02](../02-llm/README.md#7-как-обучают-llm) |
| **Semantic search** | Поиск по смыслу, а не по точному совпадению слов | «авто» находит «машину» | [04](../04-rag/README.md#3-кирпичики-rag-embeddings-и-векторная-бд) |
| **SKILL.md** | Файл-точка входа навыка: YAML-метаданные (`name`, `description`) + инструкции | См. `.agents/skills/*/SKILL.md` | [03](../03-agents/README.md#5-skills--переиспользуемые-наборы-навыков) |
| **Skill (навык)** | Папка с `SKILL.md`: процедурное знание «как делать класс задач» + ресурсы | Навык заполнения PDF-форм, навык draw.io | [03](../03-agents/README.md#5-skills--переиспользуемые-наборы-навыков) |
| **System prompt** | Системный промпт — задаёт постоянную роль и рамки модели/агента | «Ты — вежливый ассистент службы поддержки» | [02](../02-llm/README.md#8-промпт-и-его-части) |
| **Temperature** | Параметр случайности при выборе токена | 0 — код, 0.7 — текст, 1.2 — креатив | [02](../02-llm/README.md#5-как-llm-генерирует-ответ) |
| **Tool (инструмент)** | Функция, которую агент может вызвать | Веб-поиск, SQL-запрос, чтение файла | [03](../03-agents/README.md#2-инструменты-и-tool-calling) |
| **Tool calling** | Механизм, которым LLM запрашивает вызов инструмента | `{ "tool": "get_weather", "args": {...} }` | [03](../03-agents/README.md#2-инструменты-и-tool-calling) |
| **top-K** | Сколько ближайших фрагментов вернуть при поиске | top-K = 5 | [04](../04-rag/README.md#5-этап-запроса-online-rag-пайплайн) |
| **Transformer** | Архитектура нейросети, на которой построены LLM | GPT, BERT, T5 | [02](../02-llm/README.md#4-трансформер-и-механизм-внимания) |

| Термин | Кратко | Примеры | Подробно |

| **Индексация** | Подготовка базы знаний для RAG: загрузка → чанкинг → эмбеддинг → сохранение | Загрузка 100 PDF в векторную БД | [04](../04-rag/README.md#4-этап-индексации-offline) |
| **Контекстное окно** | Максимум токенов, которые модель учитывает за один запрос | GPT-4o — 128K, Claude — 200K | [02](../02-llm/README.md#6-контекстное-окно) |
| **Модель** | Функция с обучаемыми параметрами | Линейная регрессия, GPT-4o, ResNet | [01](../01-foundations/README.md#2-что-такое-модель) |
| **Мультиагентная система** | Несколько агентов, совместно решающих задачу | Агент-планировщик + агент-кодер + агент-тестер | [03](../03-agents/README.md#9-мультиагентные-системы) |
| **Нейрон** | Простой вычислительный элемент: взвешенная сумма + активация | `y = activation(w1·x1 + w2·x2 + b)` | [01](../01-foundations/README.md#3-нейросеть-простыми-словами) |
| **Нейросеть** | Гибкая функция из множества нейронов, организованных в слои | CNN, Transformer | [01](../01-foundations/README.md#3-нейросеть-простыми-словами) |
| **Оркестратор** | Главный агент, раздающий подзадачи исполнителям | Ведущий агент координирует подагентов | [03](../03-agents/README.md#9-мультиагентные-системы) |
| **Пайплайн** | Последовательность шагов: выход одного — вход следующего | Загрузка → чанкинг → эмбеддинг → поиск | [05](../05-pipelines-frameworks/README.md#1-что-такое-пайплайн) |
